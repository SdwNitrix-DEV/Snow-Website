Privacy Policy

Effective Date: 04 April 2026

Last Updated: 08 April 2026

1. Introduction

This Privacy Policy applies to the Discord bot "Snow" (hereinafter referred to as "the Bot") and its related services. By adding the Bot to your server or interacting with it, you consent to the data collection and usage practices described in this policy.

2. Data We Collect

To provide its core functionalities, the Bot collects and stores the following specific data points:

  • Discord User IDs: Used to link saved characters to specific users and to log moderation actions (e.g., timeouts, kicks, bans).
  • Discord Server (Guild) IDs and Channel IDs: Used to configure server-specific settings, such as moderation log channels and music queues.
  • Image URLs: When a user uploads an image for a custom character avatar, the Discord attachment URL is stored.
  • Transient Message Data: The Bot requires the "Message Content Intent" to read and execute text commands. Message content is processed temporarily in memory to execute commands and is not persistently stored, except when explicitly saved as a custom character name or recorded in a server's configured moderation log.
  • Support Ticket Transcripts (Support Lodge Only): When a user opens a support ticket in our official server, a full HTML transcript of the conversation is generated and archived for staff review.

3. How We Use Your Data

The collected data is used exclusively for the operational functionality of the Bot. This includes:

  • Storing and retrieving user-created characters via the character vault.
  • Enforcing server moderation rules and logging moderator actions.
  • Managing active music queues for specific servers.
  • Providing expected command responses.
  • Providing dedicated user support and archiving ticket transcripts to resolve disputes and track bug reports.
  • Actively deleting user command triggers (such as the -say prefix command) to keep chat channels clean, if granted the "Manage Messages" permission by a server administrator.

4. Data Storage and Security

Data is stored securely in an internal SQLite database hosted on our secure servers. We implement standard security measures to protect against unauthorized access, alteration, or destruction of this data.

5. Data Sharing and Disclosure

We do not sell, rent, or trade your data to any third parties. Data is only accessible to the Bot's developer and is only disclosed if required by law or to comply with Discord's Developer Terms of Service.

6. Data Retention and Deletion

  • User Control: Users can delete their stored character data at any time using the /char_delete command.
  • Server Control: Server administrators can clear server-specific settings (like mod-logs) by disabling the feature via commands.
  • Right to Erasure: If you wish to have all your data manually removed from the database, you may contact the developer directly.
